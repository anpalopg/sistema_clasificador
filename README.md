# Descripción del Proyecto

**Sistemas Clasificadores** es un proyecto de planificación y gestión orientado al desarrollo de un sistema capaz de organizar y clasificar información mediante una estructura de trabajo organizada.

El proyecto se desarrolla aplicando herramientas de **gestión de proyectos**, permitiendo organizar:

- Actividades
- Responsabilidades del equipo
- Cronograma de trabajo
- Presupuesto del proyecto
- Seguimiento del progreso

La planificación del proyecto se realiza mediante herramientas como **WBS, Matriz RACI, cronograma y control de actividades**.

---

#  Objetivo del Proyecto

Planificar y estructurar el desarrollo del proyecto **Sistemas Clasificadores** mediante herramientas de gestión de proyectos que permitan:

- Definir las tareas necesarias para el desarrollo
- Organizar el trabajo del equipo
- Controlar tiempos y recursos
- Establecer responsabilidades
- Realizar seguimiento del progreso del proyecto

#  Gestión del Proyecto

La gestión del proyecto se basa en la organización de tareas y responsabilidades para asegurar el cumplimiento de los objetivos.

Las principales actividades de gestión incluyen:

- Definición del objetivo del proyecto
- Definición del alcance
- Asignación de roles
- Planificación de tareas
- Seguimiento del progreso

---

#  Matriz RACI

La **Matriz RACI** permite identificar las responsabilidades dentro del proyecto.

| Rol | Significado |
|----|----|
| **R (Responsible)** | Responsable de ejecutar la actividad |
| **A (Accountable)** | Responsable final del resultado |
| **C (Consulted)** | Persona consultada durante la actividad |
| **I (Informed)** | Persona informada del progreso |

Esta matriz permite mejorar la organización del equipo y evitar confusión en las responsabilidades.

#  Estructura del Proyecto (WBS)

El proyecto se organiza mediante una **Work Breakdown Structure (WBS)** que divide el trabajo en partes más pequeñas y manejables.

### Fases principales del proyecto

## 1. Gestión del proyecto
- Definir objetivo del proyecto
- Definir alcance
- Asignar roles del equipo

## 2. Planificación
- Elaboración del cronograma
- Identificación de tareas
- Definición de recursos

## 3. Desarrollo
- Diseño del sistema clasificador
- Implementación del sistema
- Integración de componentes

## 4. Seguimiento
- Monitoreo del progreso
- Control de tareas
- Evaluación del avance

## 5. Cierre
- Validación del proyecto
- Documentación final

#  Cronograma del Proyecto

El cronograma permite planificar las actividades del proyecto de acuerdo con el tiempo disponible.

El cronograma incluye:

- Fecha de inicio de actividades
- Fecha de finalización
- Duración de tareas
- Dependencias entre actividades

Esto permite tener control sobre el progreso del proyecto.

---

#  Gestión de Actividades

Las actividades del proyecto se registran mediante una **lista de tareas**, que permite controlar:

- Nombre de la actividad
- Responsable
- Estado de la actividad
- Progreso

Estados posibles de las actividades:

- No iniciado
- En progreso
- Completo
- Atrasado
- En espera

---

#  Presupuesto del Proyecto

El presupuesto del proyecto permite estimar los recursos necesarios para su desarrollo.

Incluye:

- Recursos humanos
- Recursos técnicos
- Costos asociados al desarrollo del proyecto

El presupuesto permite llevar un control financiero del proyecto.
